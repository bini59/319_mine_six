# PRD: 지뢰찾기 도박형 변형 게임 (가칭: Contract Mines)

- **버전**: v0.1 (프로토타입 스펙)
- **작성일**: 2026-07-21
- **상태**: Draft

---

## 1. 개요

### 1.1 한 줄 정의

클래식 지뢰찾기의 연역 게임성을 유지하면서, **배율/캐시아웃**과 **구역 계약(Contract Zone)** 시스템으로 도박적 긴장감을 극대화한 싱글플레이 웹 게임.

### 1.2 배경

지뢰찾기는 본질적으로 "정보를 하나씩 열며 리스크를 관리하는 게임"으로, 도박 메커니즘과 구조적 궁합이 좋다 (선례: Stake Mines). 본 프로젝트는 순수 연역 게임성을 훼손하지 않는 선에서 도박적 재미를 극대화하는 것을 목표로 한다.

### 1.3 디자인 원칙 (설계 헌법)

1. **숫자는 절대 거짓말하지 않는다.** 모든 불확실성은 규칙으로 명시된다. 은폐된 랜덤 금지.
2. **플레이어는 자신의 죽음을 납득할 수 있어야 한다.** 모든 리스크는 플레이어가 스스로 선택한 것이어야 한다.
3. **이미 있는 암묵적 구조를 명시적 시스템으로 꺼낸다.** 외부 이식보다 내재 구조 증폭 우선.
4. **완급 리듬을 존중한다.** 작업 구간(빠른 오픈)과 사고 구간(정밀 추론)의 교차를 파괴하는 기능은 코어에 넣지 않는다.

## 2. 목표 및 비목표

### 2.1 목표

- 클래식 지뢰찾기 대비 "한 판 더" 루프 강도 향상 (세션당 판 수 증가)
- 플레이어 스스로 리스크 수준을 설계하는 자기 결정형 난이도 시스템
- 파훼 순간의 강렬한 연출로 성취감 극대화

### 2.2 비목표 (v0.1에서 하지 않는 것)

- 실제 화폐/암호화폐 베팅 (가상 포인트만 사용)
- 멀티플레이어, 리더보드
- 모바일 네이티브 앱 (웹 반응형까지만)
- 계정 시스템 / 서버 저장 (로컬 저장만)

## 3. 코어 게임 루프

```
[판 시작]
  ├─ 1단계: 보드 설정 (크기, 지뢰 수) → 베이스 배율 결정
  ├─ 2단계: 전역 제약 선택 (선택사항) → 배율 가산
  ├─ 3단계: 플레이
  │    ├─ 칸 오픈 시마다 배율 누적 상승
  │    ├─ 언제든 [구역 계약] 체결 가능 (공간 베팅)
  │    └─ 언제든 [캐시아웃] 가능 (수익 확정 후 판 종료)
  ├─ 지뢰 클릭 → 전액 손실, 판 종료
  └─ 전체 클리어 or 캐시아웃 → 수익 확정
[즉시 리트라이]
```

**3층 베팅 구조**: ① 판 전체 제약(베이스 배율) → ② 구역 계약(공간 베팅) → ③ 캐시아웃 타이밍(진행 베팅)

## 4. 기능 명세

### 4.1 기본 지뢰찾기 (P0)

- 클래식 규칙 준수: 첫 클릭 지뢰 면제, 숫자 = 인접 지뢰 수, 깃발, 코드(chord) 오픈
- 보드 프리셋: 9x9/10지뢰, 16x16/40지뢰, 16x30/99지뢰 + 커스텀
- **숫자는 항상 진실** (미믹은 구역 계약 내부에서만 예외적으로 허용, 4.3 참조)

### 4.2 배율 & 캐시아웃 시스템 (P0)

- 판 시작 시 베팅 포인트 설정
- 안전 칸 오픈마다 배율 상승. 상승 곡선은 남은 안전 칸 대비 지뢰 밀도 기반 (후반일수록 가파름)
- 지뢰 밀도를 플레이어가 설정 → 밀도 높을수록 배율 곡선 가파름 (리스크 셀렉션)
- **캐시아웃 버튼 상시 노출**: 현재 확정 가능 금액 실시간 표시
- 지뢰 클릭 시 베팅액 전액 손실
- 논리적으로 확정 불가능한 찍기 국면(50/50)에서 캐시아웃이 유의미한 결정이 되도록 UI에서 현재 배율/다음 배율을 명확히 대비

### 4.3 구역 계약 (Contract Zone) (P0 — 본작의 핵심 차별점)

- **체결**: 판 시작 전 또는 진행 중, 드래그로 직사각 영역 지정 + 제약 1개 선택
- **타이밍 배율**: 시작 전(정보 없음) 체결 시 고배율, 진행 중(정보 있음) 체결 시 저배율 — 체결 시점의 공개 정보량에 반비례
- **파훼 판정**: 구역 내 모든 안전 칸 오픈 시 파훼 성공 → 계약 배율 즉시 적용 + 파훼 연출
- **시각 표현**: 계약 구역은 보라색 테두리로 상시 표시 (시각적 압박 유지)
- **중첩 계약**: 구역 중첩 허용. 중첩 칸의 제약은 동시 적용, 배율은 곱연산
- **파기(손절)**: 진행 중 계약 포기 가능. 페널티 = 현재 누적 배율의 일부 차감 (파라미터, 초기값 30%)
- **미믹의 격리**: 미믹 계약 체결 시 "거짓 숫자 정확히 1개, 이 구역 안에만 존재"를 보장 → 구역 밖 숫자는 100% 신뢰 유지, 연역 게임성 보존

### 4.4 제약 카탈로그 (P0: 4종으로 시작)

| 제약 | 분류 | 규칙 | 배율(초기값) | 채택 근거 |
|---|---|---|---|---|
| 무깃발 | 정보 | 구역 내 깃발 금지 | +0.4x | 정보 해소에 기억 부하, 본질 정합 |
| 블라인드 숫자 | 정보 | 구역 내 숫자 3초 후 페이드아웃 | +0.5x | 상동 |
| 미믹 계약 | 정보 | 구역 내 거짓 숫자 정확히 1개 | +1.2x | 규칙화된 거짓 = 고난도 연역 |
| 지뢰 밀도 업 | 구조 | 구역 내 지뢰 추가 배치(체결이 시작 전일 때만) | +0.2x/개 | 내재 구조 증폭 |

**P1 후보 (코어 검증 후)**: 편도 티켓, 육각 보드, 노 세컨드찬스
**보류**: 타임어택 — 사고 구간 리듬 파괴. 넣더라도 변두리 옵션, 코어 금지.

### 4.5 확률 스캔 아이템 (P1)

- 확률 자동 표기는 기본 제공하지 않음 (계산의 재미 보존)
- 대신 소모성 아이템 "스캔 1회권": 지정 칸 1개의 지뢰 확률 표시. 포인트로 구매 → 사용 여부 자체가 베팅

### 4.6 파훼 연출 (P0 — 간소화 버전)

- 구역 마지막 안전 칸 오픈 순간: 짧은 슬로모(0.3s) → 구역 전체 황금 물결 애니메이션 → 배율 카운트업 팝 + 사운드
- v0.1은 CSS 애니메이션 + Web Audio 수준. 파티클 라이브러리 도입은 P1
- 중첩 계약 동시 파훼 시 연출 강화 (콤보 표기)

### 4.7 동적 밸런싱 (P2)

- 제약별 클리어율 로컬 수집 → 클리어율이 목표 범위를 벗어나면 배율 자동 보정
- v0.1은 정적 배율 + 수동 튜닝. 텔레메트리 스키마만 미리 설계

## 5. UX 요구사항

- **캐시아웃 버튼**: 항상 보이되, 오클릭 방지를 위해 홀드-투-컨펌 (0.5s)
- **배율 HUD**: 현재 배율 / 다음 칸 성공 시 배율 / 확정 가능 금액 3종 상시 표시
- **계약 HUD**: 활성 계약 목록, 각 계약의 남은 안전 칸 수, 파기 버튼
- **죽음 화면**: 어떤 선택(제약/계약/오픈)이 사망으로 이어졌는지 리플레이 요약 표시 — "납득" 원칙의 UI적 구현
- 데스크톱 우선, 모바일은 드래그 구역 지정 대체 입력(두 모서리 탭) 제공

## 6. 기술 스펙 (프로토타입)

- **스택**: React + TypeScript, 상태는 Zustand, 스타일 Tailwind, 빌드 Vite
- **저장**: localStorage (포인트 잔액, 통계, 설정)
- **보드 엔진**: UI와 분리된 순수 함수 모듈 (`generateBoard`, `openCell`, `resolveContract`, `calcMultiplier`) — 단위 테스트 대상
- **배율 공식 초안**: `stepMultiplier = 1 / P(safe)` 기반에 하우스 팩터 곱 (파라미터화)
- **성능**: 16x30 보드에서 오픈 연쇄(flood fill) 60fps 유지

## 7. 성공 지표 (프로토타입 기준)

- 셀프 플레이테스트에서 세션당 평균 판 수 ≥ 클래식 지뢰찾기 대비 1.5배
- 구역 계약 사용률 ≥ 판당 50% (안 쓰이면 코어 실패 신호)
- 사망 후 30초 내 리트라이 비율 ≥ 70% ("납득" 지표 프록시)

## 9. 리스크 및 대응

| 리스크 | 대응 |
|---|---|
| 미믹이 구역 안에서도 리듬을 무너뜨림 | 미믹 구역 최대 크기 제한 (예: 6x6) |
| 배율 공식의 기댓값 헛점 | 엔진 분리 덕에 몬테카를로 시뮬레이션으로 사전 검증 |
| 중첩 계약 배율 곱연산 인플레 | 중첩 수 상한(2) + 곱연산에 감쇠 계수 |
| 손절 페널티가 약해 비가역성 붕괴 | 파기 페널티를 30%에서 시작해 상향 테스트 |
| 도박 모사 게임의 윤리 문제 | 실화폐 미연동 원칙 고정, 가상 포인트 무료 리필 |
